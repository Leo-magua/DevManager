## 快速入门

### 第一步：访问看板

在浏览器中打开：
```
http://your-server-ip:81
```

### 第二步：选择项目

点击顶部下拉菜单，选择一个项目查看详情。

### 第三步：开始使用

- 查看任务看板了解当前进度
- 提交新需求到需求池
- 点击任务卡片更新状态

## 操作指南

### 如何提交新需求

1. 确保已选择项目
2. 在右侧"提交新需求"区域填写信息：
   - **标题**: 简洁描述需求（必填）
   - **描述**: 详细说明（可选）
   - **优先级**: 高/中/低
   - **类别**: 功能/Bug/前端/后端/工具
3. 点击"提交需求"按钮
4. 成功后会在需求池显示

### 如何更新任务状态

1. 在看板中找到要更新的任务卡片
2. 点击卡片，状态会循环切换：
   ```
   Pending → In_Progress → Completed → Pending
   ```
3. 状态变更会自动记录到日志

### 如何切换项目

1. 点击顶部导航栏的"选择项目"下拉菜单
2. 从列表中选择一个项目
3. 页面会自动加载该项目的所有数据

### 如何刷新数据

**自动刷新**: 页面每10秒自动刷新一次

**手动刷新**: 
- 点击项目信息卡片的"刷新"按钮
- 或重新选择项目

## 最佳实践

### 1. 项目初始化

对于每个新项目，建议创建以下文件：

```
NewProject/
├── dev_state.json          # 开发状态（DevManager读取）
├── user_backlog.json       # 需求池（DevManager写入）
├── .kimi/
│   └── status.md          # 项目状态（人工维护）
└── README.md              # 项目说明
```

### 2. 需求管理流程

```
提出需求 → 评估优先级 → 开发中 → 测试 → 完成
    ↑                                      ↓
    └──────────── 反馈修复 ←───────────────┘
```

使用 DevManager 跟踪每个阶段的状态。

### 3. 与 AI Agent 配合

1. **设置当前任务**: 在代码中更新 `dev_state.current_context`
2. **记录试错**: Agent 每次重试时增加 `trial_count`
3. **错误追踪**: 出错时写入 `last_error`，看板会显示

示例代码（Node.js）：
```javascript
const fs = require('fs');

function updateAgentStatus(taskId, taskName, error = null) {
  const data = JSON.parse(fs.readFileSync('dev_state.json', 'utf-8'));
  
  data.current_context = {
    agent_task_id: taskId,
    task_name: taskName,
    start_time: new Date().toISOString(),
    trial_count: error ? (data.current_context.trial_count + 1) : 0,
    last_error: error
  };
  
  fs.writeFileSync('dev_state.json', JSON.stringify(data, null, 2));
}

// 开始任务
updateAgentStatus('TASK-001', '实现登录功能');

// 出错重试
updateAgentStatus('TASK-001', '实现登录功能', '数据库连接失败');
```

### 4. 定期回顾

建议每周进行一次：

1. 查看"已完成"列，总结本周成果
2. 检查"需求池"，整理和排序待办
3. 更新技术栈和项目描述
4. 清理过期的日志记录

### 5. 团队协作

多人使用时：

- 每个人通过 DevManager 查看项目状态
- 提交需求时注明提交人（可在描述中添加）
- 状态变更会自动同步给所有人

## 故障排查

### 问题1: 页面空白，显示"选择项目"

**原因**: 没有配置监控的项目

**解决**: 编辑 `config.json` 添加项目，或访问 `/api/projects` 查看扫描到的项目

### 问题2: 任务状态无法更新

**原因**: 
1. 项目目录没有写权限
2. dev_state.json 格式错误

**解决**:
```bash
# 检查权限
ls -la /var/www/AllProject/YourProject/dev_state.json

# 修复权限
chmod 644 /var/www/AllProject/YourProject/dev_state.json
```

### 问题3: 新需求提交失败

**原因**: 网络问题或服务未启动

**解决**:
```bash
# 检查服务状态
ps aux | grep "node server.js"

# 检查端口
ss -tlnp | grep 81

# 查看错误日志
tail -f /var/www/AllProject/DevManager/server.log
```

## 高级用法

### 自定义刷新间隔

编辑 `config.json`:
```json
{
  "features": {
    "auto_refresh": 30  // 改为30秒
  }
}
```

### 扩展数据字段

可以在 `dev_state.json` 中添加自定义字段，前端会自动显示：

```json
{
  "custom_fields": {
    "sprint": "Sprint-3",
    "milestone": "Beta版本"
  }
}
```

### 集成 CI/CD

在部署脚本中更新状态：

```bash
# 部署前
curl -X POST http://localhost:81/api/agent-status \
  -d '{"task_id": "DEPLOY", "task_name": "正在部署..."}'

# 部署后
curl -X POST http://localhost:81/api/agent-status \
  -d '{"task_id": null, "task_name": "部署完成"}'
```
