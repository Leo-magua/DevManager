# AllProject DevManager v2.0 - 智能开发指挥中心

> 从静态看板进化为真正的开发指挥中心，点击即开发，自然语言即需求

---

## 🎯 v2.0 核心升级

### 1. 点击卡片 = 开始实际开发

**之前**: 点击看板卡片仅切换前端状态  
**现在**: 
1. 点击"待处理"卡片 → 弹出确认 → 加入开发队列
2. Kimi CLI 通过 `devman-cli claim` 认领任务
3. Agent 实际开发完成后 → 自动更新为"已完成"

### 2. 自然语言需求输入

**之前**: 手动填写标题、描述、优先级  
**现在**: 
```
输入: "添加一个暗黑模式切换功能，放在页面右上角"
     ↓ AI解析
输出: 
  标题: 添加暗黑模式切换功能
  优先级: Medium
  类别: Frontend
  自动创建任务到看板
```

## 🚀 快速开始

### 启动服务

```bash
cd /var/www/AllProject/DevManager
nohup node server-v2.js > server-v2.log 2>&1 &
```

### 安装CLI工具

```bash
chmod +x bin/devman-cli
sudo ln -s /var/www/AllProject/DevManager/bin/devman-cli /usr/local/bin/devman-cli
```

### 访问看板

```
http://your-server-ip:81
```

## 📋 使用流程

### 场景1: 通过看板启动开发

1. 打开看板 http://180.184.99.247:81
2. 选择项目
3. 点击"待处理"列的卡片
4. 确认"开始开发"
5. 卡片状态变为"开发中"
6. Kimi CLI 自动获取任务并执行
7. 开发完成后自动变为"已完成"

### 场景2: 自然语言提需求

1. 在看板底部找到"自然语言需求输入"
2. 输入: "添加用户登录功能，使用JWT认证，高优先级"
3. 点击"预览解析结果"查看AI解析
4. 点击"提交需求"
5. 需求自动创建为看板卡片

### 场景3: Kimi CLI 工作流

```bash
# 1. 查看当前任务
devman-cli next

# 2. 认领任务（开始开发）
devman-cli claim

# 3. 执行开发...
# ... 编写代码 ...

# 4. 标记完成
devman-cli done "实现了用户登录API，包含JWT token生成和验证"

# 5. 如果出错，报告并选择是否重试
devman-cli error "数据库连接失败" --retry
```

## 🔌 API 列表

### 任务队列 API

| 端点 | 方法 | 说明 |
|-----|------|------|
| `/api/queue` | GET | 获取队列状态 |
| `/api/queue/claim` | POST | Agent认领任务 |
| `/api/queue/complete` | POST | 标记任务完成 |
| `/api/queue/error` | POST | 报告错误 |
| `/api/agent/next-task` | GET | 获取当前任务（给Agent用）|

| 端点 | 方法 | 说明 |
|-----|------|------|
| `/api/projects/:id/features/:fid/start` | POST | 点击卡片开始开发 |

## 🛠️ CLI 命令详解

### devman-cli status

查看当前队列状态：
```
📊 队列状态
  待处理: 3
  已完成: 12

▶️  正在执行
  任务: 实现用户登录API
  ID: TASK-1712745600000
```

### devman-cli next

查看下一个待处理任务：
```
📋 当前任务
  名称: 实现用户登录API
  描述: 使用JWT认证的登录接口
  优先级: High

💡 提示: 使用 'devman-cli claim' 认领此任务
```

### devman-cli claim

认领并开始执行任务：
```
✅ 任务已认领!
  任务: 实现用户登录API
  ID: TASK-1712745600000

📝 开始开发吧！完成后运行: devman-cli done '完成说明'
```

### devman-cli done "说明"

完成任务并更新状态：
```
✅ 任务已完成!
  任务: 实现用户登录API
  说明: 实现了用户登录API，包含JWT token生成和验证

💡 还有 2 个任务等待，运行 'devman-cli next' 查看
```

### devman-cli error "错误信息" [--retry]

报告错误（可选重试）：
```
⚠️  错误已记录 (重试次数: 1)
  错误: 数据库连接失败

🔄 已设置重试，请修复后重新执行
```

### devman-cli nlp "需求描述" [项目ID]

提交自然语言需求：
```
🤖 正在解析自然语言需求...
  输入: 添加暗黑模式切换功能

📋 解析结果
  标题: 添加暗黑模式切换功能
  优先级: Medium
  类别: Frontend

确认创建此需求? (y/n)
```
